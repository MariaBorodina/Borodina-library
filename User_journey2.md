# User Journey: Author Path for MVP Online Library

## Stage 1: Author Login and Home Page Arrival
- **User goal**: Access account and navigate to author dashboard
- **Actions**: Logs in via Sign in/Sign up, lands on homepage, notices "My Books" in navigation menu
- **Touchpoints**: Login form, navigation menu with "My Books" link, homepage content
- **Emotions**: Anticipation, sense of ownership, professional purpose
- **Pain points**: None if login is smooth; friction if authentication is complex
- **Improvements**: Social login for authors; persistent login; clear visual indicator for "My Books"

## Stage 2: Navigating to My Books Section
- **User goal**: Access personal book catalog and management interface
- **Actions**: Clicks "My Books" in navigation menu
- **Touchpoints**: Navigation menu item, page transition, author dashboard header
- **Emotions**: Purpose, readiness to manage work, curiosity about current catalog
- **Pain points**: None if navigation is clear; confusion if "My Books" is hard to find
- **Improvements**: Highlight active menu item; show book count badge on "My Books" icon

## Stage 3: Viewing Author's Book Catalog
- **User goal**: See overview of all authored books and their status
- **Actions**: Views grid/list of books with covers, titles, status indicators (draft/published/complete)
- **Touchpoints**: Book cards (cover, title, status badge, action buttons), empty state message if no books
- **Emotions**: Pride in body of work, satisfaction seeing catalog, motivation to add more
- **Pain points**: None if catalog loads quickly; frustration if books don't display correctly
- **Improvements**: Sort/filter options (by date, status, title); show recent activity; quick stats

## Stage 4: Initiating New Book Creation
- **User goal**: Start process of adding a new book to the library
- **Actions**: Clicks "Add New Book" button (prominent CTA)
- **Touchpoints**: "Add New Book" button, form overlay/page transition, form title
- **Emotions**: Excitement, creative anticipation, sense of possibility
- **Pain points**: None if button is prominent; hesitation if process seems complex
- **Improvements**: Button animation on hover; placeholder example; clear call-to-action text

## Stage 5: Filling Book Information
- **User goal**: Provide essential metadata for the new book
- **Actions**: Fills in title, author (pre-filled), description, genre/realm, cover image upload, publication date
- **Touchpoints**: Form fields (text inputs, textarea, image upload, dropdowns, date picker), save/cancel buttons
- **Emotions**: Focused creativity, satisfaction as details come together, slight pressure to get it right
- **Pain points**: Form fatigue if too many fields; confusion about required vs optional fields
- **Improvements**: Real-time preview; auto-save draft; clear required field indicators; genre suggestions

## Stage 6: Saving Book Information
- **User goal**: Persist book metadata and move to content management
- **Actions**: Clicks "Save Book Info" button after completing form
- **Touchpoints**: Primary action button, loading state, success message/transition
- **Emotions**: Accomplishment, relief, readiness to add content
- **Pain points**: Anxiety if save fails; frustration if validation errors aren't clear
- **Improvements**: Inline validation; clear error messages; optimistic UI; undo option

## Stage 7: Accessing Book Content Management
- **User goal**: Begin adding/uploading content for the book
- **Actions**: Navigates to "Content" tab or section for the newly created book
- **Touchpoints**: Content management interface, chapter/increment list, "Add Content" button
- **Emotions**: Engagement, anticipation of sharing work, focus on delivery
- **Pain points**: None if transition is smooth; confusion if content section isn't obvious
- **Improvements**: Clear tab labels; visual indicator of content progress; quick-add options

## Stage 8: Uploading Book Content Increment
- **User goal**: Add a chapter or section of book content
- **Actions**: Clicks "Add New Chapter/Section", uploads file or pastes text, sets title, saves
- **Touchpoints**: Content upload form, file picker/text editor, title field, save button, progress indicator
- **Emotions**: Satisfaction as content is added, sense of progress, creative fulfillment
- **Pain points**: Technical issues with upload; formatting problems; unclear content structure
- **Improvements**: Drag-and-drop upload; rich text editor; auto-format options; chapter preview

## Stage 9: Viewing Content Increments
- **User goal**: Review and manage existing content sections
- **Actions**: Views list of uploaded chapters/sections with titles, order, edit/delete options
- **Touchpoints**: Content list items (title, order number, date, action buttons), reorder controls
- **Emotions**: Organization satisfaction, ability to see work structure, pride in progress
- **Pain points**: None if list is clear; frustration if reordering is difficult
- **Improvements**: Drag-and-drop reordering; content preview on hover; word/page count per section

## Stage 10: Editing Book Information
- **User goal**: Update metadata for an existing book (title, description, etc.)
- **Actions**: Clicks "Edit Info" on a book card, modifies fields, saves changes
- **Touchpoints**: Edit form (pre-filled with current data), field inputs, save/cancel buttons
- **Emotions**: Control over work, satisfaction in keeping information current, minor effort
- **Pain points**: None if edit is straightforward; frustration if changes don't save properly
- **Improvements**: Inline editing for simple fields; change history; version notes

## Stage 11: Editing Content Increment
- **User goal**: Modify existing chapter/section content
- **Actions**: Clicks "Edit" on a content item, makes changes in editor, saves
- **Touchpoints**: Content editor (text area/rich text), editing tools, save button, preview option
- **Emotions**: Refinement satisfaction, attention to detail, commitment to quality
- **Pain points**: None if editor is good; frustration if formatting is lost or save fails
- **Improvements**: Version history for content; side-by-side compare; autosave; export/import options

## Stage 12: Removing Content Increment
- **User goal**: Delete a chapter/section that is no longer needed
- **Actions**: Clicks "Delete" on content item, confirms deletion in prompt
- **Touchpoints**: Delete button, confirmation modal, confirmation message
- **Emotions**: Decisiveness, slight melancholy (removing work), relief if content was problematic
- **Pain points**: Anxiety about accidental deletion; frustration if recovery isn't possible
- **Improvements**: Undo option after deletion; archive instead of delete; clear confirmation text

## Stage 13: Removing a Book Entirely
- **User goal**: Delete a book and all its content from the library
- **Actions**: Clicks "Delete Book" on book card, confirms deletion in prompt
- **Touchpoints**: Delete button (possibly red/destructive), confirmation modal, final confirmation
- **Emotions**: Serious decision, potential sadness, closure if retiring work
- **Pain points**: High anxiety about permanent loss; fear of accidental deletion
- **Improvements**: Multiple confirmation steps; archive option; clear warning about permanence; recovery period

## Stage 14: Publishing/Updating Book Status
- **User goal**: Make book available to readers or update its publication status
- **Actions**: Changes status from draft to published (or updates published version)
- **Touchpoints**: Status dropdown/toggle, publish button, status indicator, notification
- **Emotions**: Accomplishment, vulnerability (sharing work), pride, anticipation of readers
- **Pain points**: None if process is clear; anxiety if unsure about publishing implications
- **Improvements**: Preview before publishing; scheduled publishing; clear status explanations; reader notification options

## Stage 15: Viewing Reader Engagement (Optional)
- **User goal**: See how readers are interacting with their work
- **Actions**: Navigates to stats/analytics section for a book, views reads, likes, comments
- **Touchpoints**: Analytics dashboard, metrics display (reads, completion rate, likes), time filters
- **Emotions**: Validation, connection with audience, motivation to create more, curiosity
- **Pain points**: None if stats are meaningful; disappointment if engagement is low; privacy concerns
- **Improvements**: Reader comments (if enabled); geographic data; reading duration; exportable reports
