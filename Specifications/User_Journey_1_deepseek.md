## Happy Path User Journey – MVP: Free Online Library (Fantasy & Sci-Fi)

### Stage 1: Landing on Home Page
| Field | Description |
|-------|-------------|
| **User goal** | Understand what the web app offers and whether it matches their interest in fantasy/Sci-Fi books |
| **Actions** | Opens the web app URL. Sees hero section with compelling tagline, background art, and call-to-action (e.g., “Start exploring”) |
| **Touchpoints** | Home page, hero section |
| **Emotions** | Curious, hopeful, visually engaged |
| **Pain points** | None (hero loads fast, genre clear) |
| **Improvements** | None needed for MVP |

### Stage 2: Browsing by Category (Realm)
| Field | Description |
|-------|-------------|
| **User goal** | Explore books based on thematic “realms” (e.g., “Dragon Realms”, “Cyberpunk Wastelands”) |
| **Actions** | Clicks on “Browse by Realm” in navigation. Scrolls through realm cards |
| **Touchpoints** | Browse by Realm page |
| **Emotions** | Excited, imaginative |
| **Pain points** | None if realms are clearly labeled and distinct |
| **Improvements** | Add visual icons per realm post-MVP |

### Stage 3: Selecting a Realm
| Field | Description |
|-------|-------------|
| **User goal** | Filter books by a specific realm they love (e.g., “Time Travel Archives”) |
| **Actions** | Clicks on a realm card. Sees list of book covers + titles from that realm |
| **Touchpoints** | Realm detail page |
| **Emotions** | Focused, intrigued |
| **Pain points** | None |
| **Improvements** | None |

### Stage 4: Spotting an Interesting Book
| Field | Description |
|-------|-------------|
| **User goal** | Find a book that sparks their fancy |
| **Actions** | Scans book covers and titles. Clicks on a cover that looks appealing |
| **Touchpoints** | Book info page |
| **Emotions** | Anticipation, curiosity |
| **Pain points** | None if cover and title are clear |
| **Improvements** | Hover effect with short teaser post-MVP |

### Stage 5: Reading Book Info Page
| Field | Description |
|-------|-------------|
| **User goal** | Decide if they want to read the book (synopsis, author, length, tags) |
| **Actions** | Reads synopsis, checks author name, sees “Start reading” button |
| **Touchpoints** | Book info page |
| **Emotions** | Engaged, informed |
| **Pain points** | None if synopsis is well-written |
| **Improvements** | Add content warnings or ratings later |

### Stage 6: Opening the Book for Paginated Reading
| Field | Description |
|-------|-------------|
| **User goal** | Begin reading immediately without obstacles |
| **Actions** | Clicks “Start reading”. Book opens in paginated view (no login required) |
| **Touchpoints** | Reading page |
| **Emotions** | Satisfied, immersed |
| **Pain points** | None – frictionless access |
| **Improvements** | Remember last page per device post-MVP |

### Stage 7: Reading First Few Pages
| Field | Description |
|-------|-------------|
| **User goal** | Get hooked by the story |
| **Actions** | Swipes or clicks to turn pages. Reads comfortably |
| **Touchpoints** | Reading page |
| **Emotions** | Relaxed, entertained |
| **Pain points** | None |
| **Improvements** | Font size toggle later |
